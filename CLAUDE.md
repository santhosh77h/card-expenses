## Project Overview

**Vector** — a privacy-first credit card statement parser. *Your Money. Directed.* Expo React Native frontend + FastAPI backend that extracts transactions from PDF statements using LLM-powered parsing with regex fallback.

## Commands

### Mobile (Expo SDK 51 + Dev Client)

```bash
cd mobile
npm start                # Metro bundler in dev-client mode
npx expo run:ios         # Build + run on iOS (first run prebuilds native dirs)
npx expo run:android     # Build + run on Android
npx expo prebuild --clean  # Regenerate ios/ and android/ from app.json
```

Native dirs (`ios/`, `android/`) are generated by prebuild and gitignored. First build compiles native C++ (op-sqlite), subsequent runs use `npm start`.

### Backend

```bash
cd backend
pip install -r requirements.txt
uvicorn app:app --reload --host 0.0.0.0 --port 8000
```

Requires `OPENAI_API_KEY` env var for LLM parsing; falls back to regex without it. Configuration via `.env` file or environment variables (see `.env.example`).

## Architecture

### Data Flow — Write-Through Pattern

SQLite (op-sqlite) is the source of truth. Zustand holds an in-memory copy.

- **On mutation**: write to SQLite first → update Zustand state
- **On startup**: `initDatabase()` → Zustand hydrates cards/activeCardId from AsyncStorage → `_hydrateSqlite()` loads transactions/statements/enrichments/monthlyUsage from SQLite → `dbReady` gate allows rendering

### Backend API

Single endpoint: `POST /parse-statement/json` — accepts PDF file + optional password, returns transactions, summary, CSV, detected bank/currency, card info. Max 10MB (configurable via `MAX_FILE_SIZE_MB`).

LLM parser uses OpenAI structured output (`gpt-4o-mini` default) with Pydantic models. Falls back gracefully to regex if OpenAI unavailable.

### Multi-Currency

Supported: INR, USD, EUR, GBP. Each card/statement/transaction tracks currency. `formatCurrency()` in store handles locale-specific grouping (Indian vs Western).

## op-sqlite API Notes

- Use `executeSync()` for synchronous queries (returns `QueryResult` with `.rows` as `Array<Record<string, Scalar>>`)
- `execute()` is async
- Rows accessed directly as `result.rows` (not `result.rows._array`)
- For sync transactions, use `BEGIN`/`COMMIT`/`ROLLBACK` with `executeSync` (not the async `transaction()` method)
- **Version constraint**: Must use op-sqlite ≤11.x with Expo SDK 51 (RN 0.74). Versions 12+ use `invokeAsync(jsi::Runtime&)` which requires RN 0.76+.

## Conventions

- **State mutations**: Always write to SQLite first, then call the Zustand setter. See existing patterns in `store/index.ts`.
- **Categories**: 12 fixed categories. Backend source of truth: `backend/app/categories.py` (keywords, colors, icons). Frontend mirror: `mobile/src/theme.ts`. Keep them in sync.
- **TypeScript path aliases**: `@/*` → `src/*`, plus `@components/*`, `@screens/*`, `@store/*`, `@utils/*`, `@navigation/*`
- **Dark theme only**: Background `#0A0E1A`, accent `#00E5A0`
- **Babel**: requires `react-native-reanimated/plugin` as last plugin
- **Schema changes**: Any change to the SQLite schema (new table, new column, altered column, new index) **must** follow `MIGRATION_GUIDE.md`. Never modify existing migrations — only append new ones. Bump `LATEST_VERSION` in `schema.ts` and add a new numbered migration function.
